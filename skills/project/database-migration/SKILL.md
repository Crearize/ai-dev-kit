# Database Migration Skill - DBマイグレーション

## 禁止事項（最重要）

> **以下は絶対に禁止。違反は許容されない。**

- **既存マイグレーションファイルの変更・削除禁止**: チェックサムエラーが発生する
- **mainブランチ確認なしでのファイル作成禁止**: バージョン番号が重複する
- **バージョン番号の欠番禁止**: 連番で管理

## ファイル命名規則

プロジェクトのマイグレーションツール（Flyway, Prisma, Alembic 等）の命名規則に従う。

## バージョン番号決定手順（必須）

### Step 1: mainの最新状態を取得

```bash
git pull origin main
```

### Step 2: 最新バージョン番号を確認

プロジェクトのマイグレーションディレクトリを確認し、最新のバージョン番号を取得する。

### Step 3: マイグレーションファイル作成

次の連番でファイルを作成。

## SQL記述ルール

- `CREATE TABLE IF NOT EXISTS` を使用（冪等性の確保）
- インデックスは必要なカラムに適切に作成
- 外部キー制約を適切に設定
- デフォルト値を明示的に指定
