# Document Review Guide

**Note**: Review only when document files (documents/**, *.md) have been changed.

## Required Reference Documents

1. `documents/development/development-policy.md` - Development guidelines

---

## Review Checklist

### 1. Document Consistency

- [ ] No contradictions between related documents
- [ ] Feature specifications and coding rules are consistent
- [ ] Database design documents and ER diagrams match
- [ ] New error codes added to error code list (if applicable)

### 2. Document Structure

- [ ] Heading levels (#, ##, ###) appropriate
- [ ] Table of contents logically structured
- [ ] Sections properly organized
- [ ] Markdown formatting correct

### 3. Content Accuracy

- [ ] Technical descriptions accurate
- [ ] Code examples work correctly
- [ ] Command examples executable
- [ ] No broken links

### 4. Readability

- [ ] Natural and readable prose
- [ ] Technical terms properly explained
- [ ] Lists and tables used appropriately
- [ ] Code blocks have proper language specification

### 5. Completeness

- [ ] All necessary information included
- [ ] Implementation examples sufficient
- [ ] Edge cases and caveats documented

### 6. CLAUDE.md / .cursorrules Consistency

- [ ] References in configuration files are correct
- [ ] Checklists are up to date
- [ ] New documents referenced from configuration (if needed)

### 7. Feature Specifications (if applicable)

- [ ] Feature overview clear
- [ ] Use cases specific
- [ ] API specifications clear
- [ ] Data model clear

### 8. Database Design (if applicable)

- [ ] Table definitions clear
- [ ] Column definitions (types, constraints) accurate
- [ ] Index design documented
- [ ] Foreign key constraints clear

### 9. Coding Rules (if applicable)

- [ ] Rules specific and actionable
- [ ] Good/bad examples provided
- [ ] Prohibited patterns clearly stated
